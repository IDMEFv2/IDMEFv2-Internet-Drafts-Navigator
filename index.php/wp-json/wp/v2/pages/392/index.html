      <\/span>Introduction<\/span><\/h2>\n\n\n\n

      IDMEFv2 stands for Incident Detection Message Exchange Format.<\/p>\n\n\n\n

      IDMEFv2 defines a format to describe cyber and\/or physical incidents or events suspected to participate to an incident (events of interest).<\/p>\n\n\n\n

      Here are few examples of IDMEFv2 use case:<\/p>\n\n\n\n

        \n
      • a virus has been detected in a mail,<\/li>\n\n\n\n
      • an intruder is trying to enter a building,<\/li>\n\n\n\n
      • a server is not responding probably down,<\/li>\n\n\n\n
      • a recon scan has been detected on the web server,<\/li>\n\n\n\n
      • an abnormal temperature has been detected in the datacenter and server might have stopped,<\/li>\n\n\n\n
      • multiple failed authentication on the financial server in the middle of the night,<\/li>\n\n\n\n
      • a drone has been detected flying around the building,<\/li>\n\n\n\n
      • a high temperature wave is predicted next week which might cause power outage,<\/li>\n\n\n\n
      • etc.<\/li>\n<\/ul>\n\n\n\n

        An IDMEFv2 message is composed of classes (Alert, Analyzer, Sensor, Source, Target, \u2026) and attributes ( CreateTime, StartTime, ID, IP, user, protocol, Location, GeoLocation, \u2026)<\/p>\n\n\n\n

        IDMEFv2 can be used in cyber detection management system (Anti-virus, Firewall, SIEM (Security Information & Event Management), ..) and in physical detection management system (CCTV, Badger, movement sensor, PSIM (Physical Security Information Management), \u2026). It can also be used in combined environment with cyber and physical detection.<\/p>\n\n\n\n

        This universal and unique format allows to analyse and correlate multiple types of incidents together and detect complex and\/or combined incidents and\/or attacks. IDMEFv2 is a response to the security of IoT(Internet of Things) and IIoT (Industrial IoT) and all type of \u201cSmart\u201d Architectures.<\/p>\n\n\n\n

        IDMEFv2 can be describe in JSON and transported over HTTPs<\/p>\n\n\n\n

        <\/span>IDMEFv2 Use case<\/span><\/h2>\n\n\n\n
        IDMEFv2 is a universal format for cyber and physical systems.<\/p>\n\n\n\n
        It has been defined in a consortium composed of academic and industrial incident detection experts. The first draft has been deployed and tested in real scale on ground segment infrastructure on a large international research project during two years,  the 7Shield project<\/a>. IDMEFv2 is well fitted for critical infrastructure and sensitive site.<\/p>\n\n\n\n
        By combining cyber and physical incidents as well as natural hazards, IDMEFv2 shoud also be a good choice for smart systems and particularly for autonomous vehicles.<\/p>\n\n\n\n
        <\/span>IDMEFv2 evolution<\/span><\/h2>\n\n\n\n
